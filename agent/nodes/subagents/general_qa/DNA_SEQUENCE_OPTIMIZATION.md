# DNA序列分析与基因表达问题优化文档

## 优化概述

根据日志分析中发现的三个关键案例错误，对`general_qa`节点的prompt进行了针对性优化，主要解决以下问题：

1. **DNA序列六框翻译错误**：反向互补链计算不正确，ORF识别混淆
2. **DNA转录翻译错误**：起始密码子定位错误，翻译结果矛盾
3. **基因表达问题错误**：基因名称混淆（Braveheart vs Nkx2-5），表达模式错误

## 优化内容

### 1. 在`get_complete_inference_prompt`中添加DNA序列分析规则

#### 规则1：六框翻译准确性（Six-Frame Translation Accuracy）
- **要求**：
  - 正向框：从位置1、2、3开始翻译
  - 反向互补框：**必须先计算反向互补序列**（A↔T, C↔G，然后反转），再从反向互补序列的位置1、2、3开始翻译
  - **禁止**：不计算反向互补序列就直接翻译反向框
  - **验证**：每个框的翻译必须使用标准遗传密码表逐密码子验证

#### 规则2：起始密码子（ATG）识别
- **要求**：
  - 必须逐位置扫描：从位置1、2、3等开始检查每个3碱基窗口
  - 必须验证精确匹配：ATG必须作为连续碱基出现（如位置13-15：A-T-G，而不是位置13、15、17：A-T-G跨密码子分割）
  - **禁止**：不进行逐位置验证就假设ATG位置
  - **验证**：说明确切位置范围（如"ATG在位置32-34"）

#### 规则3：转录准确性（Transcription Accuracy）
- **要求**：
  - 碱基配对规则：A→U, T→A, C→G, G→C（DNA到RNA）
  - 必须转录整个序列：对DNA序列中的每个碱基应用规则
  - **验证**：验证DNA中的ATG在mRNA中变为AUG

#### 规则4：翻译准确性（Translation Accuracy）
- **要求**：
  - 必须从识别的起始密码子开始：翻译从AUG密码子开始（从识别的ATG位置）
  - 必须使用标准遗传密码表：每个密码子必须在标准遗传密码表中查找
  - 必须按三联体读取：每个密码子精确读取3个碱基，无移码
  - **禁止**：不先验证起始密码子位置就翻译
  - **验证**：根据密码子表验证每个氨基酸（如AUG→Met(M), GUU→Val(V)）

#### 规则5：ORF识别（ORF Identification）
- **要求**：
  - ORF定义：必须包含ATG起始密码子**和**TAA/TAG/TGA终止密码子
  - **禁止**：将任何阅读框称为"ORF"而不包含起始和终止密码子
  - **必须说明**：如果不存在完整ORF，说明"未找到完整ORF"并解释原因
  - **对于SNP修饰的ORF**：如果问题要求含特定SNP的ORF，必须：
    - 首先识别包含目标密码子的阅读框
    - 应用SNP修饰创建修饰序列
    - 验证修饰序列是否仍形成有效ORF（如需要）

#### 规则6：寡核苷酸设计（Oligo Design）
- **要求**：
  - 必须覆盖SNP位点：Oligo序列必须包含修饰的SNP位置
  - 标准参数：长度18-25nt，GC含量40-60%，使用Wallace规则计算Tm
  - 特异性：Oligo必须区分野生型和修饰序列
  - **验证**：说明oligo序列、长度、GC含量和Tm值

### 2. 在`get_complete_inference_prompt`中添加基因表达与基因名称准确性规则

#### 规则1：基因名称验证（Gene Name Verification）
- **要求**：
  - 必须验证基因身份：在使用关于基因的知识之前，必须验证：
    1. 是蛋白质编码基因还是非编码RNA（lncRNA、miRNA等）？
    2. 正确的基因符号/名称是什么？（如Braveheart = Bvht，**不是**Nkx2-5）
  - **禁止**：混淆具有相似名称或功能的不同基因
  - **验证**：如果不确定，说明"基因身份不明确"并使用工具验证

#### 规则2：表达模式准确性（Expression Pattern Accuracy）
- **要求**：
  - 必须引用特定细胞类型：说明确切的细胞类型（如"胚胎干细胞"vs"分化心肌细胞"）
  - 必须说明方向："增加"、"减少"、"不变"或"低/高表达"
  - **禁止**：没有方向的通用陈述，如"表达变化"
  - **验证**：表达模式必须匹配特定基因的已知生物学（如Bvht在ESC中低，在心肌细胞中高）

#### 规则3：知识来源验证（Knowledge Source Verification）
- **要求**：
  - 如果检索到的知识与结论矛盾：
    1. **必须重新验证**：检查基因名称/身份是否正确
    2. **必须检查表达方向**：验证表达是增加还是减少
    3. **必须与问题对齐**：表达模式必须匹配问题的细胞类型
    4. **如果检测到不一致**：标记"inference_status": "Inference-GeneNameMismatch"并纠正

### 3. 在`get_initial_inference_prompt`中添加基因知识匹配规则

#### Hard Rule 11：基因名称与表达模式验证
- **要求**：
  1. **必须验证基因身份**：如果问题提到基因名称（如"Braveheart"），必须验证：
     - 是蛋白质编码基因还是非编码RNA？
     - 正确的基因符号是什么？（如Braveheart = Bvht，**不是**Nkx2-5）
     - **禁止**：使用关于具有相似名称/功能的不同基因的知识
  2. **必须验证表达方向**：对于表达模式问题：
     - 说明确切方向："增加"、"减少"、"不变"、"低表达"、"高表达"
     - 将表达模式匹配到问题中提到的特定细胞类型
     - **禁止**：没有方向的通用陈述，如"表达变化"
  3. **必须标记不匹配**：如果检索到的知识提到错误的基因或错误的表达模式：
     - 将match_confidence标记为"Low"
     - 添加到match_reasoning："基因名称不匹配：[检索到的基因] vs [问题中的基因]"或"表达模式不匹配：[检索到的模式] vs [问题中的模式]"

### 4. 在`get_answer_generation_prompt`中添加序列验证规则

#### 序列问题的关键验证
- **要求**：
  1. **必须根据遗传密码验证**：如果从DNA/RNA输出氨基酸序列，使用标准遗传密码表验证每个密码子翻译
  2. **必须验证起始密码子**：如果序列以Met(M)开头，验证它对应于AUG/ATG
  3. **必须验证长度**：氨基酸序列长度应匹配 (DNA长度 - 起始位置) / 3（向下取整）
  4. **禁止**：输出与core_conclusion中描述的转录/翻译过程不匹配的序列
  5. **验证**：在输出final_answer之前，验证序列与closed_inference_path中的翻译步骤匹配

## 预期效果

这些优化规则将确保：

1. **DNA序列分析准确性**：
   - 正确计算反向互补序列
   - 准确识别起始密码子位置
   - 严格按照标准遗传密码表翻译
   - 正确区分阅读框和ORF

2. **基因表达问题准确性**：
   - 正确识别基因身份（蛋白质编码 vs 非编码RNA）
   - 避免基因名称混淆（如Braveheart vs Nkx2-5）
   - 准确描述表达模式方向

3. **答案验证**：
   - 序列答案与推理步骤一致
   - 翻译结果可验证
   - 表达模式与知识匹配
