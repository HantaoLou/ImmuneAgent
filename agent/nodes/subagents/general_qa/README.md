# 生物医学问题解题系统的LangGraph节点构成（含分支逻辑与交互关系）
基于补充优化方案（数值计算、专业算法、跨领域知识库、文本型答案处理），LangGraph节点设计遵循「**功能单一化、流程模块化、分支精细化**」原则，共划分12个核心节点，覆盖全场景问题处理，节点间通过「条件判断边」实现动态流程跳转。


## 一、LangGraph节点总览（含功能定位与输入输出）
| 节点ID | 节点名称                          | 核心功能定位                                  | 输入数据（来自前序节点）                          | 输出数据（传递至后序节点）                          |
|--------|-----------------------------------|-----------------------------------------------|---------------------------------------------------|---------------------------------------------------|
| N0     | 输入预处理与问题分类              | 清洗输入、判定问题类型与数据完整性             | 原始问题文本（含选项/实验数据）                   | 清洗后文本+问题类型标签+数据完整性标签             |
| N1     | 问题拆解与领域定位                | 提取客观条件/研究目的，定位核心领域            | N0输出（不含数值计算/算法类标签的问题）           | 结构化条件字典+核心领域列表+研究目标               |
| N2     | 计算/算法需求识别                  | 识别数值计算/专业算法需求，提取计算相关特征    | N0输出（含数值计算/算法类标签的问题）             | 计算类型标签（数值/算法）+ 关键参数/公式线索       |
| N3     | 跨领域知识检索                    | 调用分领域知识库，获取基础+专业知识点          | N1输出（领域列表）/ N2输出（算法领域）            | 领域-知识点对照表+知识有效性标签（有效/缺失）      |
| N4     | 计算步骤拆解与公式匹配            | 分解计算逻辑，匹配生物医学专属公式             | N2输出（数值计算类）+ N3输出（计算相关知识）      | 计算步骤列表+匹配公式（含参数定义）+ 单位换算规则  |
| N5     | 算法参数提取与适用性验证          | 提取算法输入参数，验证算法与问题的匹配性       | N2输出（算法类）+ N3输出（算法知识库）            | 算法参数字典+适用性结果（适用/不适用）+ 备选算法   |
| N6     | 初次关联推理                      | 建立实验现象/概念与知识的静态匹配              | N1输出（结构化条件）+ N3输出（基础知识点）        | 现象-知识匹配表+匹配可信度标签（高/中/低）         |
| N7     | 完整逻辑推理（含动态/计算推导）   | 整合知识完成构象/机制推理，或计算步骤推导      | N6输出（推理类）/ N4输出（数值类）/ N5输出（算法类） | 闭合推理通路（含计算过程）+ 核心结论              |
| N8     | 多类型答案生成                    | 按问题类型生成专属格式答案（选择/数值/文本）   | N7输出（核心结论）+ N0输出（问题类型标签）        | 结构化答案（选项匹配表/数值结果/文本答案）         |
| N9     | 结果校验与一致性判断              | 校验答案与推理通路的一致性，评估结果可靠性     | N8输出（结构化答案）+ N7输出（推理通路）          | 一致性标签（一致/不一致）+ 可靠性评分（1-5分）     |
| N10    | 知识/计算异常处理                  | 处理知识缺失、计算错误、算法不适用等异常        | N3输出（知识缺失）/ N4输出（公式匹配失败）/ N9输出（不一致） | 异常类型标签+解决方案建议（重试/人工介入）         |
| N11    | 人工介入触发                      | 输出人工介入提示，提供当前推导中间结果         | N10输出（需人工介入的异常）                       | 人工介入指南+中间结果快照（便于接续处理）          |


## 二、核心分支逻辑（节点间的边规则）
LangGraph通过「**条件判断边**」实现不同问题类型的流程分支，避免无效节点调用，核心分支逻辑如下：

### 1. 入口分支（N0→N1/N2）
- **分支条件1**：若N0输出的「问题类型标签」为「选择题/文本匹配题/机制解释题」→ 跳转至 **N1（问题拆解与领域定位）**
- **分支条件2**：若N0输出的「问题类型标签」为「数值计算题/专业算法题」→ 跳转至 **N2（计算/算法需求识别）**
- **异常分支**：若N0输出的「数据完整性标签」为「严重缺失」→ 直接跳转至 **N10（知识/计算异常处理）**


### 2. 推理/计算分支（N1/N2→后续节点）
#### （1）非计算类分支（N1→N3→N6→N7）
- N1完成领域定位后，跳转至 **N3（跨领域知识检索）**；
- 若N3输出「知识有效性标签」为「有效」→ 跳转至 **N6（初次关联推理）**；
- 若N3输出「知识有效性标签」为「缺失」→ 跳转至 **N10（异常处理）**；
- N6完成静态匹配后，跳转至 **N7（完整逻辑推理）**（侧重机制/概念推导）。

#### （2）数值计算分支（N2→N3→N4→N7）
- N2识别数值计算需求后，跳转至 **N3（跨领域知识检索）**（获取计算相关公式）；
- N3检索完成后，跳转至 **N4（计算步骤拆解与公式匹配）**；
- 若N4输出「公式匹配结果」为「匹配成功」→ 跳转至 **N7（完整逻辑推理）**（侧重计算过程推导）；
- 若N4输出「公式匹配结果」为「匹配失败」→ 跳转至 **N10（异常处理）**。

#### （3）专业算法分支（N2→N3→N5→N7）
- N2识别算法需求后，跳转至 **N3（跨领域知识检索）**（获取算法知识库）；
- N3检索完成后，跳转至 **N5（算法参数提取与适用性验证）**；
- 若N5输出「适用性结果」为「适用」→ 跳转至 **N7（完整逻辑推理）**（侧重算法执行推导）；
- 若N5输出「适用性结果」为「不适用」→ 跳转至 **N10（异常处理）**（尝试备选算法）。


### 3. 结果校验与异常分支（N7→N8→N9→N10/N11）
- N7完成推理/计算后，跳转至 **N8（多类型答案生成）**；
- N8输出结构化答案后，跳转至 **N9（结果校验与一致性判断）**；
- **分支条件1**：若N9输出「一致性标签」为「一致」且「可靠性评分≥4分」→ 流程结束，输出最终答案；
- **分支条件2**：若N9输出「一致性标签」为「不一致」或「可靠性评分≤3分」→ 跳转至 **N10（异常处理）**；
- **分支条件3**：若N10输出「异常类型标签」为「无法自动解决」（如核心知识缺失）→ 跳转至 **N11（人工介入触发）**；
- **分支条件4**：若N10输出「异常类型标签」为「可重试解决」（如公式匹配偏差）→ 回溯至对应节点（如N4/N5）重新处理。


## 三、关键节点的核心执行逻辑（适配补充优化功能）
### 1. N2（计算/算法需求识别）：精准区分计算类型
- **数值计算识别规则**：检测问题中的「计算关键词」（calculate/compute/determine）+「单位特征」（mol/L/%/kb）+「数学符号」（θ/Σ/√）；
- **算法需求识别规则**：匹配「算法关键词」（Watterson估计器/多基因评分/序列比对）+「分析目标」（遗传率估算/抗体亲和力预测）。

### 2. N3（跨领域知识检索）：覆盖19个子领域
- **知识检索优先级**：1级（领域专属知识库，如遗传学→遗传定律库）→ 2级（跨领域关联知识，如分子生物学+遗传学）→ 3级（通用基础库）；
- **缺失处理**：若某领域知识缺失，自动调用「领域替代知识」（如免疫学缺失时，调用基础医学相关知识）。

### 3. N4（计算步骤拆解与公式匹配）：生物医学公式专属处理
- **步骤拆解逻辑**：按「问题目标→核心变量→公式选择→参数代入→单位换算→结果验证」拆解；
- **公式库覆盖**：含遗传率计算、浓度换算、PCR产物量估算等120+生物医学专属公式，标注公式适用场景（如哈迪-温伯格公式仅适用于随机交配群体）。

### 4. N5（算法参数提取与适用性验证）：专业算法适配
- **参数提取逻辑**：从问题中提取「必填参数」（如Watterson估计器需「样本数」「多态位点数」）+「可选参数」（如置信区间）；
- **适用性验证**：检查「问题场景」与「算法适用范围」匹配度（如群体遗传学算法不适用临床样本数据）。

### 5. N8（多类型答案生成）：按类型定制格式
- **选择题**：输出「选项匹配表」，标注排除理由（如"与计算结果矛盾"）；
- **数值题**：输出「计算步骤+最终结果+单位」，标注精度控制（如保留2位小数）；
- **文本匹配题**：输出「True/False判断+逻辑验证过程」或「术语填空+匹配依据」。


## 四、容错机制设计（节点级异常处理）
| 异常场景                | 触发节点 | 处理逻辑                                                                 |
|-------------------------|----------|--------------------------------------------------------------------------|
| 知识缺失（某领域知识点无） | N3       | 跳转N10，尝试调用「领域替代知识」，若失败则触发N11人工介入               |
| 公式匹配失败            | N4       | 跳转N10，输出「备选公式建议」（如浓度计算可备选质量浓度/摩尔浓度公式）    |
| 算法不适用              | N5       | 跳转N10，自动匹配「备选算法」（如多基因评分不适用时，调用GWAS关联分析）  |
| 推理通路不一致          | N9       | 跳转N10，回溯至N6重新匹配知识，或调整推理逻辑（如补充中间推理步骤）      |
| 计算结果异常（如负值）  | N7       | 跳转N10，检查「参数代入错误」或「公式适用场景」，重新推导                |
